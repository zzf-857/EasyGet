# EasyGet 手动打包说明

这份文档用于记录 `EasyGet` 的手动打包流程。当前项目是 `WPF + .NET 8`，最稳妥的打包方式是使用 `dotnet publish` 生成 `win-x64` 自包含发布目录。

## 1. 打包前准备

确认本机具备以下条件：

- Windows 系统
- 已安装 `.NET 8 SDK`
- 在项目根目录打开终端：
  `E:\AIMadeupTools\01_DesktopApps\EasyGet`

可先执行下面命令确认 SDK 是否正常：

```powershell
dotnet --version
```

## 2. 标准打包命令

在项目根目录执行：

```powershell
dotnet publish .\EasyGet.csproj -c Release -r win-x64 --self-contained true
```

说明：

- `-c Release`：使用正式发布配置
- `-r win-x64`：面向 64 位 Windows
- `--self-contained true`：把 .NET 运行时一起打进去，目标机器不用额外安装 .NET

## 3. 打包产物位置

打包完成后，产物目录在：

```text
E:\AIMadeupTools\01_DesktopApps\EasyGet\bin\Release\net8.0-windows\win-x64\publish\
```

其中最重要的是：

- `EasyGet.exe`
- 以及同目录下的全部 `dll`、运行时文件、多语言资源目录

## 4. 发给别人时怎么处理

不要只发 `EasyGet.exe`，要把整个 `publish` 文件夹一起打包。

推荐做法：

1. 打包完成后，进入 `publish` 目录。
2. 选中里面全部文件和文件夹。
3. 压缩成一个 zip 包。
4. 把 zip 包发给别人。

对方使用时：

1. 解压整个 zip。
2. 直接双击 `EasyGet.exe` 运行。

## 5. 首次运行的注意事项

程序首次运行时会自动检查并下载以下工具：

- `yt-dlp`
- `ffmpeg`

它们不会放在 `publish` 目录里，而是默认下载到：

```text
%LocalAppData%\EasyGet\tools
```

也就是通常类似：

```text
C:\Users\你的用户名\AppData\Local\EasyGet\tools
```

这意味着：

- 如果目标电脑可以联网，首次启动后程序会自动补齐这些工具。
- 如果目标电脑不能联网，需要先在一台能联网的机器上运行一次程序，然后把 `%LocalAppData%\EasyGet\tools` 整个目录一起拷过去。

## 6. 配置和历史记录位置

程序运行后还会在本地生成这些数据：

- 配置文件：`%LocalAppData%\EasyGet\config.json`
- 下载历史：`%LocalAppData%\EasyGet\history.db`
- 崩溃日志：`publish\logs\`

如果只是发安装包给别人，一般不需要带上这些文件。

## 7. 我自己每次打包时建议这样做

建议固定按下面顺序执行：

```powershell
dotnet restore
dotnet publish .\EasyGet.csproj -c Release -r win-x64 --self-contained true
```

打完后检查：

- `publish` 目录是否生成
- `EasyGet.exe` 是否存在
- 本机是否能正常双击启动

## 8. 当前项目已验证的命令

这个项目已经实际验证过下面这条命令可以正常出包：

```powershell
dotnet publish -c Release -r win-x64 --self-contained true
```

输出目录为：

```text
bin\Release\net8.0-windows\win-x64\publish\
```

## 9. 常用备注

- 如果只是“给 Windows 64 位用户直接用”，继续用 `win-x64` 就行。
- 如果后面要做安装包，建议再额外接入 `Inno Setup` 或 `MSIX`；目前这个项目还没有安装器脚本。
- 发布过程中如果看到 `WFAC010` 这一类 High DPI 警告，目前不会阻止出包。
