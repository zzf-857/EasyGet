# EasyGet 手动打包说明

这份文档用于记录 `EasyGet` 的手动打包流程。当前项目是 `WPF + .NET 8`，最稳妥的打包方式是使用 `dotnet publish` 生成 `win-x64` 自包含发布目录。

## 1. 打包前准备

确认本机具备以下条件：

- Windows 系统
- 已安装 `.NET 8 SDK`
- 在项目根目录打开终端：
  `E:\AIMadeupTools\01_DesktopApps\EasyGet`

可先执行下面命令确认 SDK 是否正常：

```powershell
dotnet --version
```

## 2. 推荐打包脚本

在项目根目录执行：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\publish-win-x64.ps1
```

脚本会依次执行：

- `dotnet restore`
- `dotnet test`
- `dotnet publish`
- 检查发布目录中的 `EasyGet.exe`
- 生成 zip 包

只想做发布 smoke 检查、不生成 zip 时，可以执行：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\publish-win-x64.ps1 -SkipZip
```

常用参数：

- `-Configuration Release`：使用正式发布配置
- `-Runtime win-x64`：面向 64 位 Windows
- `-SkipTests`：跳过测试，仅用于已经单独跑过测试的场景
- `-SkipZip`：只生成发布目录，不压缩 zip
- `-Version 1.1.7`：本次发布写入程序集的版本号，必须与 `EasyGet.csproj` 中的 `<Version>` 一致

需要构建安装包时，先安装 Inno Setup 6，然后执行：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\build-installer.ps1 -Version 1.1.7
```

该脚本会同时生成 `EasyGet-Setup-vX.Y.Z.exe`、`EasyGet-win-x64-Release.zip` 和 `easyget-update.json`。

## 3. 手动打包命令

在项目根目录执行：

```powershell
dotnet publish .\EasyGet.csproj -c Release -r win-x64 --self-contained true
```

说明：

- `-c Release`：使用正式发布配置
- `-r win-x64`：面向 64 位 Windows
- `--self-contained true`：把 .NET 运行时一起打进去，目标机器不用额外安装 .NET

## 4. 打包产物位置

脚本默认输出目录在：

```text
artifacts\publish\Release\win-x64\
```

手动 `dotnet publish` 的默认产物目录在：


```text
E:\AIMadeupTools\01_DesktopApps\EasyGet\bin\Release\net8.0-windows\win-x64\publish\
```

其中最重要的是：

- `EasyGet.exe`
- 以及同目录下的全部 `dll`、运行时文件、多语言资源目录

## 5. 发给别人时怎么处理

不要只发 `EasyGet.exe`，要把整个 `publish` 文件夹一起打包。

推荐做法：

1. 打包完成后，进入 `publish` 目录。
2. 选中里面全部文件和文件夹。
3. 压缩成一个 zip 包。
4. 把 zip 包发给别人。

对方使用时：

1. 解压整个 zip。
2. 直接双击 `EasyGet.exe` 运行。

## 6. 首次运行的注意事项

程序首次运行时会自动检查并下载以下工具：

- `yt-dlp`
- `ffmpeg`

它们不会放在 `publish` 目录里，而是默认下载到：

```text
%LocalAppData%\EasyGet\tools
```

也就是通常类似：

```text
C:\Users\你的用户名\AppData\Local\EasyGet\tools
```

这意味着：

- 如果目标电脑可以联网，首次启动后程序会自动补齐这些工具。
- 如果目标电脑不能联网，需要先在一台能联网的机器上运行一次程序，然后把 `%LocalAppData%\EasyGet\tools` 整个目录一起拷过去。

## 7. 配置和历史记录位置

程序运行后还会在本地生成这些数据：

- 配置文件：`%LocalAppData%\EasyGet\config.json`
- 下载历史：`%LocalAppData%\EasyGet\history.db`
- 崩溃日志：`publish\logs\`

如果只是发安装包给别人，一般不需要带上这些文件。

## 8. 我自己每次打包时建议这样做

建议固定按下面顺序执行：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\publish-win-x64.ps1
```

打完后检查：

- `publish` 目录是否生成
- `EasyGet.exe` 是否存在
- 本机是否能正常双击启动

## 9. 当前项目已验证的命令

这个项目已经实际验证过下面这条命令可以正常出包：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\publish-win-x64.ps1 -SkipZip
```

输出目录为：

```text
artifacts\publish\Release\win-x64\
```

## 10. 常用备注

- 如果只是“给 Windows 64 位用户直接用”，继续用 `win-x64` 就行。
- 正式分发优先使用 `EasyGet-Setup-vX.Y.Z.exe` 安装包；便携分发使用 `EasyGet-win-x64-Release.zip`。
