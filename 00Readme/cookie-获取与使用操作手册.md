# 智能登录与 Cookie 使用手册

## 默认用法：直接下载

多数情况下无需导出、复制或粘贴 Cookie：

1. 在“视频下载”或“批量下载”中输入链接。
2. EasyGet 先匿名解析公开内容。
3. 平台明确要求登录时，EasyGet 自动尝试本机已安装浏览器的登录配置。
4. 如果还没有登录，在设置页点击“登录并检测”，EasyGet 会使用系统默认浏览器打开目标平台。
5. 在平台官网完成登录或验证码；EasyGet 会在三分钟内自动回查对应平台的登录 Cookie 元数据并更新状态。
6. 检测到登录后，下载任务会优先使用匹配的浏览器配置。

“无感”表示尽量复用本机已有登录状态，不表示绕过登录、验证码、地区限制、付费权限或平台服务条款。

## 支持的浏览器

Windows 版会自动发现下列浏览器的默认配置和多个用户配置：

- Chrome
- Edge
- Firefox
- Brave
- Chromium
- Vivaldi
- Opera / Opera GX

EasyGet 只枚举可用配置并交给 yt-dlp 尝试，不在设置页显示 Cookie 值或完整配置路径。设置页的登录检测只读取 Cookie 域名、名称和有效期，不读取 Cookie 值；浏览器数据库被占用或某个配置不可用时，下载会自动尝试下一个配置。

## 设置页状态

打开“系统设置 → 智能登录与 Cookie”可以：

- 开启或关闭智能 Cookie 模式；
- 刷新浏览器配置与最近认证健康状态；
- 使用系统默认浏览器按平台登录并自动检测；
- 清除单个平台或全部 EasyGet 兼容登录会话；
- 在折叠的高级区域手动导入 Cookie。

状态含义：

- 绿色：该平台近期已有真实下载或解析验证成功的策略；
- 主题色：已从浏览器 Cookie 元数据检测到该平台登录状态，下载时会优先读取对应配置；
- 黄色：已检查可读浏览器配置，但未检测到该平台登录 Cookie；
- 灰色：发现了浏览器配置，但浏览器数据库暂时无法读取，下载时仍会继续尝试。

## 系统浏览器登录

第三方平台官网不会把 Cookie 直接回调给任意桌面软件。EasyGet 点击“登录并检测”后采用本地轮询形成闭环：

1. 使用 Windows 系统默认浏览器打开平台真实登录页。
2. 最多等待三分钟，并周期性检查本机浏览器 Cookie 数据库中的域名、名称和有效期。
3. 只要发现该平台的有效登录标记，设置页立即显示检测到的浏览器；不会复制、显示或记录 Cookie 值。
4. 正式下载时，匹配的浏览器配置排在其他配置之前，由 yt-dlp 读取实际 Cookie。
5. 如果三分钟内没有检测到，登录不会失效；完成登录后点击设置页顶部“刷新”即可再次检测，也可以直接尝试下载。

## 兼容登录

只有系统浏览器数据库因占用、加密策略或权限而无法读取时，才建议使用“兼容登录”。兼容窗口直接打开目标平台的真实网页，EasyGet 不读取、保存或记录账号、密码和验证码：

1. 确认窗口顶部显示的平台和允许域名正确。
2. 在网页内正常完成登录或验证。
3. EasyGet 只导出当前平台允许域名的会话 Cookie，并以临时文件交给 yt-dlp。
4. 登录完成后点击“已完成登录，继续”；检测到平台会话后窗口关闭，所有等待该平台认证的任务继续运行。
5. 点击“取消”只取消本次认证，不清除其他平台会话。

如果 WebView2 Runtime 缺失，请按窗口提示安装 Microsoft Edge WebView2 Runtime，再重试登录。

## 高级手动导入

只有系统浏览器读取与兼容登录都不可用时才建议使用高级手动导入。

- Header 格式（如 `key=value; key2=value2`）必须先选择所属平台，否则不会写入配置。
- Netscape 或 JSON 格式若包含真实域名，EasyGet 会按域名自动拆分。
- 无关平台的 Cookie 行会被丢弃，不会复制到其他站点。
- 填写后点击“加密保存手动 Cookie”；成功后输入框会清空，内容按平台写入 Windows DPAPI 加密仓库，不以明文保存在 `config.json` 或备份中。
- 旧版本遗留且尚未选择平台的 Header Cookie 会先进入同目录的 DPAPI 加密隔离项；下次启动仍会回填到输入框，选择正确平台后再点击保存即可完成迁移。
- 关闭“智能 Cookie 模式”只停用浏览器自动探测和兼容登录，不会禁用用户主动保存的加密手动 Cookie。
- 要删除手动 Cookie，先选择平台，再点击“清空手动 Cookie”；只删除该平台，不影响其他平台。

不要在 Issue、日志、截图、聊天记录或网盘中分享 Cookie。Cookie 通常等同于临时登录凭据。

## 本地隐私与清理

| 数据 | 本地位置 | 说明 |
|---|---|---|
| 手动 Cookie | `%LocalAppData%/EasyGet/manual-cookies/` | 按平台隔离；未归属的旧版内容暂存为加密隔离项；均由当前 Windows 用户 DPAPI 加密 |
| 兼容登录会话 | `%LocalAppData%/EasyGet/sessions/<platform>/` | 每个平台独立的 WebView2 用户目录 |
| 健康记录 | `%LocalAppData%/EasyGet/cookie-health.json` | 不含 Cookie 值和完整浏览器路径 |
| 临时 Cookie 文件 | `%LocalAppData%/EasyGet/temp/cookies/` | 仅当前用户可访问；任务结束删除，启动时清理遗留 |

平台列表行内的“清除”只清理 EasyGet 托管会话和对应健康记录；高级区域的“清空手动 Cookie”只删除选中平台的加密手动内容。两者都不会退出 Chrome、Edge、Firefox 等外部浏览器中的账号。

## 常见问题

### 公开内容为什么还提示登录？

平台可能触发年龄、机器人验证、频率限制或地区策略。EasyGet 只在错误被判定为认证相关时切换 Cookie 来源；普通网络错误不会反复弹登录。

### 浏览器已经登录但仍不可用？

可能是配置数据库被占用、浏览器加密策略变化、Cookie 已过期，或登录发生在另一个用户配置。EasyGet 会继续尝试其他配置；仍不可用时可手动选择“兼容登录”。

### 为什么不再推荐 Cookie 导出扩展？

智能模式可以复用浏览器状态或使用平台真实登录页，减少手工导出、过期维护和误传 Cookie 的风险。手动导入仍作为高级兜底保留。

### 如何让某个平台重新登录？

点击该平台的“登录并检测”，在系统默认浏览器中重新登录即可。行内“清除”只清理 EasyGet 的兼容登录数据和健康记录，不会退出系统浏览器账号。
